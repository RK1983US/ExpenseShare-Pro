# Contributing to ExpenseShare Pro

Thank you for your interest in contributing to ExpenseShare Pro! We welcome contributions from the community to help make this expense management application even better.

## 🤝 How to Contribute

### Reporting Issues
- **Bug Reports**: If you find a bug, please open an issue with detailed steps to reproduce
- **Feature Requests**: Suggest new features or improvements through GitHub issues
- **Documentation**: Help improve our documentation and user guides

### Types of Contributions Welcome
- 🐛 **Bug Reports**: Help us identify and fix issues
- 📖 **Documentation**: Improve user guides, help docs, and README
- 🎨 **UI/UX Improvements**: Suggest interface enhancements
- 🌍 **Translations**: Help make the app available in more languages
- 💡 **Feature Ideas**: Propose new functionality
- 🧪 **Testing**: Help test new features and releases

## 📋 Issue Guidelines

### Bug Reports
Please include:
- **Operating System**: Windows version and architecture
- **Application Version**: Version of ExpenseShare Pro you're using
- **Steps to Reproduce**: Clear, numbered steps
- **Expected Behavior**: What should have happened
- **Actual Behavior**: What actually happened
- **Screenshots**: If applicable, add screenshots to help explain

### Feature Requests
Please include:
- **Problem Description**: What problem does this solve?
- **Proposed Solution**: How should it work?
- **Alternatives**: Any alternative solutions you've considered
- **Use Case**: When would this feature be useful?

## 🔧 Development

### Source Code
- The application is built using Python and Tkinter
- Source code is maintained privately for security reasons
- Public releases include only the compiled executable and documentation

### Dependencies
Listed in `requirements.txt`:
- tkinter (GUI framework)
- tkcalendar (date picker)
- PIL/Pillow (image handling)
- fpdf (PDF generation)
- requests (API calls)
- openpyxl (Excel export - optional)

## 📝 Documentation Contributions

- Use clear, simple language

## 🚀 Getting Started

1. **Download**: Get the latest version from releases
2. **Test**: Try the application with various scenarios
3. **Report**: Open issues for any problems you find
4. **Suggest**: Propose improvements through issues
5. **Document**: Help improve our guides and documentation

## 📞 Support

### Self-Help Resources
- **README.md**: Complete user guide
- **docs/USER_GUIDE.md**: Step-by-step instructions

### Getting Help
- Check existing issues before creating new ones
- Use issue templates when available

- Provide detailed information to help us understand your request

## 🏷️ Labels

We use the following labels to organize issues:
- `bug`: Something isn't working
- `enhancement`: New feature or request
- `documentation`: Improvements or additions to documentation
- `good first issue`: Good for newcomers
- `help wanted`: Extra attention is needed
- `question`: Further information is requested

## 📄 Code of Conduct

### Our Standards
- **Be Respectful**: Treat everyone with respect and kindness
- **Be Constructive**: Provide helpful feedback and suggestions
- **Be Patient**: Allow time for responses and resolution
- **Be Collaborative**: Work together to improve the application

### Unacceptable Behavior
- Harassment, trolling, or personal attacks
- Spam or off-topic discussions
- Sharing of personal or sensitive information
- Any behavior that would be inappropriate in a professional setting
